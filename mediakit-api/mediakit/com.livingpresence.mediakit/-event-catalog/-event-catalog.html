EventCatalog

constructor(httpClient: <Error class: unknown class>, config: MediaKitConfig = MediaKitConfig.Default, maxConcurrency: Int = DEFAULT_MAX_CONCURRENCY, cacheTtl: <Error class: unknown class> = DEFAULT_CACHE_TTL, timeSource: <Error class: unknown class> = TimeSource.Monotonic, maxProbeAttempts: Int = DEFAULT_MAX_PROBE_ATTEMPTS, retryBackoff: <Error class: unknown class> = DEFAULT_RETRY_BACKOFF)

Parameters

httpClient

The Ktor client used for playlist probes.

config

SDK config; controls the event-number range and URL layout.

maxConcurrency

Bounds concurrent probes to avoid hammering the server when probing ~20 events at once.

cacheTtl

How long a successful loadEvents result is reused before the next call re-probes. Duration.ZERO disables caching (always probe).

timeSource

Monotonic clock used to measure cache staleness. Inject a kotlin.time.TestTimeSource in tests to advance time deterministically.

maxProbeAttempts

Max attempts per event on transient failures (default 2: one try + one retry). 4xx "missing" outcomes are never retried.

retryBackoff

Delay between probe retries. Set to Duration.ZERO to retry immediately.
